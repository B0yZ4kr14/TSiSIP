# TSiSIP Style Guide

## Writing

### Voice
- Professional
- Friendly
- Clear
- Helpful

### Tone
- Informal for docs
- Formal for legal
- Friendly for support

### Words
- Use simple words
- Avoid jargon
- Be direct
- Use active voice

## Code

### PHP
```php
function example(): string {
    return 'hello';
}
```

### JavaScript
```javascript
function example() {
    return 'hello';
}
```

### CSS
```css
.example {
    color: var(--primary);
}
```

## UI

### Buttons
- Primary: filled
- Secondary: outline
- Danger: red

### Forms
- Labels: above
- Errors: below
- Help text: below

### Tables
- Headers: bold
- Rows: alternating
- Actions: right

## Documentation

### Structure
- Title
- Overview
- Details
- Examples
- References

### Format
- Markdown
- Headers
- Lists
- Code blocks

## Commit Messages

### Format
```
type(scope): message
```

### Types
- feat
- fix
- docs
- test
- chore

## Naming

### Files
- lowercase
- kebab-case
- descriptive

### Variables
- camelCase for JS
- snake_case for PHP
- descriptive

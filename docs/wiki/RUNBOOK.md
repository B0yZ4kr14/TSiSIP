# TSiSIP Runbook

## Daily

### Morning
1. Check health
2. Review alerts
3. Check disk
4. Check memory

### Evening
1. Run backup
2. Review logs
3. Check metrics
4. Document issues

## Weekly

### Monday
1. Review weekend
2. Update tasks
3. Plan week

### Friday
1. Run tests
2. Update docs
3. Plan next week

## Monthly

### First Monday
1. Review metrics
2. Update dependencies
3. Security scan
4. Backup test

### Last Friday
1. Generate report
2. Review changes
3. Plan next month
4. Update roadmap

## Quarterly

1. Performance review
2. Capacity planning
3. Security audit
4. Disaster recovery test

## Incidents

### P1: System Down
1. Acknowledge
2. Investigate
3. Communicate
4. Resolve
5. Post-mortem

### P2: Major Issue
1. Assess impact
2. Apply workaround
3. Fix root cause
4. Verify

### P3: Minor Issue
1. Document
2. Schedule fix
3. Apply
4. Verify

## Maintenance

### Database
- Vacuum weekly
- Reindex monthly
- Archive quarterly

### Docker
- Update images monthly
- Prune weekly
- Audit quarterly

### Application
- Restart monthly
- Clear cache weekly
- Update config as needed

## Escalation

### Level 1
- On-call engineer
- Response: 15 min

### Level 2
- Senior engineer
- Response: 1 hour

### Level 3
- Team lead
- Response: 4 hours

### Level 4
- Management
- Response: 1 day

## 3. Key Decisions During Implementation

### AD-020-1: MI Command Whitelist in PHP
- Whitelist defined as hardcoded array in web/mi-commands.php
- Commands: ds_reload, domain_reload, get_statistics, dlg_list, dlg_end_dlg, tls_reload
- Role hierarchy enforced via numeric levels
- Rationale: Prevents command injection

### AD-020-2: Statistics Backpressure Design
- 30s auto-refresh interval with 5s cURL timeout
- Error path: freeze charts at last-known values, show warning banner

### AD-020-3: Database-Driven Dialplan
- dialplan table used instead of hardcoded rules
- Rules loaded at startup or via ds_reload MI command

### AD-020-4: Read-Only Dialog Viewer
- web/dialog.php queries dialog table directly via PDO
- No mutation endpoints

## 4. Security Posture

### Passed Requirements
| Requirement | Verification |
|---|---|
| R1 (RBAC devops) | All pages use requireRole('devops') minimum |
| R2 (CSRF) | All mutating forms include validateCsrfToken() |
| R3 (PDO prepared) | All DB queries use prepared statements |
| R4 (MI whitelist) | Whitelist rejects non-approved commands with 403 |
| R5 (Read-only dialog) | dialog.php has no POST handlers for mutation |
| R6 (TLS admin-only) | tls_reload requires admin role |
| R7 (Audit logging) | logAuditEvent() called on all success paths |
| R8 (XSS prevention) | All output wrapped in htmlspecialchars() |
| R9 (Error handling) | MI errors caught; user-friendly messages without stack traces |
| R10 (Failure audit) | Failed MI commands logged with error code and user identity |

### Resolved Security Findings

#### FINDING-SEC-020-001: CRUD Failure Audit Gap (MEDIUM) — RESOLVED
- **File**: web/dialplan.php, web/domains.php
- **Issue**: PDOException catch blocks set error but did NOT call logAuditEvent()
- **Fix**: Added logAuditEvent() in all CREATE/UPDATE catch blocks (commit 98af161)
- **Verification**: Brownfield scan confirms 4 logAuditEvent calls with result=false

#### FINDING-SEC-020-002: Missing Security Headers (MEDIUM) — RESOLVED
- **File**: web/common/header.php
- **Issue**: No CSP, X-Frame-Options, or X-Content-Type-Options headers
- **Fix**: Added 4 security headers via PHP header() calls (commit 98af161)
- **Verification**: curl -I confirms all headers present

#### FINDING-SEC-020-003: Session Hardening Gap (MEDIUM) — RESOLVED
- **File**: web/common/config.php, web/login.php
- **Issue**: session_start() without regenerate_id or strict cookie flags
- **Fix**: Added cookie_httponly, cookie_samesite=Strict, use_strict_mode=1, session_regenerate_id(true) on login (commit 98af161)
- **Verification**: Set-Cookie header inspection passes

- **Source**: Security Review FINDING-SEC-020-F01, Ripple R2
- **Files**: web/dialplan.php, web/domains.php
- **Severity**: MEDIUM
- **Constitution Gate**: Security Constitution Section 7 (Audit Events)

**Description**:
Add logAuditEvent() calls in all PDOException catch blocks for CREATE, UPDATE, and DELETE operations in dialplan.php and domains.php.

**Current State (dialplan.php)**:
```php
catch (PDOException $e) {
    $error = _('Failed to add dialplan rule: ') . $e->getMessage();
}
```

**Target State**:
```php
catch (PDOException $e) {
    logAuditEvent('DIALPLAN_CREATE', 'dialplan', $match_exp, false, [
        'reason' => $e->getMessage(),
    ]);
    $error = _('Failed to add dialplan rule: ') . $e->getMessage();
}
```

**Evidence Required**:
- Screenshot or log excerpt showing failed CREATE with audit entry

### Task R2: Security Headers for OCP (SECURITY)
- **Source**: Brownfield M2, Security Review FINDING-SEC-020-F02
- **File**: web/common/header.php (or nginx config)
- **Severity**: MEDIUM
- **Constitution Gate**: Security Constitution Section 5 (Secure-by-Design)

**Description**:
Add HTTP security headers to all OCP responses:
- Content-Security-Policy: default-src 'self'; script-src 'self' https://cdn.jsdelivr.net; style-src 'self' 'unsafe-inline'
- X-Frame-Options: DENY
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin

**Implementation Options**:
Option A: PHP header() calls in common/header.php
Option B: Nginx add_header directives (preferred for performance)

**Evidence Required**:
- curl -I output showing all headers present
- Browser dev tools screenshot confirming CSP

### Task R3: Session Hardening (SECURITY)
- **Source**: Brownfield M3, Security Review FINDING-SEC-020-F03
- **File**: web/common/config.php
- **Severity**: MEDIUM
- **Constitution Gate**: Security Constitution Section 2 (OCP Web Authentication)

**Description**:
1. Add session_regenerate_id(true) after successful authentication
2. Set session.cookie_secure based on X-Forwarded-Proto header
3. Set session.cookie_httponly = true
4. Set session.cookie_samesite = 'Strict'
5. Set session.use_strict_mode = 1

**Target State**:
```php
if (isset($_SERVER['HTTP_X_FORWARDED_PROTO']) && $_SERVER['HTTP_X_FORWARDED_PROTO'] === 'https') {
    ini_set('session.cookie_secure', '1');
}
ini_set('session.cookie_httponly', '1');
ini_set('session.cookie_samesite', 'Strict');
ini_set('session.use_strict_mode', '1');
```

**Evidence Required**:
- Set-Cookie header inspection showing Secure, HttpOnly, SameSite=Strict
- Session ID change verified after login

### Task R5: D3.js Local Asset Fallback (RELIABILITY)
- **Source**: Memory Synthesis Bug Pattern 3
- **File**: web/statistics.php
- **Severity**: LOW
- **Constitution Gate**: Architecture Constitution (Docker-first, air-gapped capability)

**Description**:
Add fallback for D3.js CDN load failure. If CDN is unreachable, load from local copy or display a graceful degradation message.

**Implementation**:
```html
<script src="https://cdn.jsdelivr.net/npm/d3@7/dist/d3.min.js"></script>
<script>
if (typeof d3 === 'undefined') {
    document.getElementById('statistics-chart').innerHTML = 
        '<p class="tsisip-text-muted">Charts unavailable — D3.js could not be loaded.</p>';
}
</script>
```

**Evidence Required**:
- Screenshot of graceful fallback when CDN is blocked

### 4.5 MEDIUM: Cap Apache Workers in OCP Container

**File:** `docker/ocp/Dockerfile` or new `docker/ocp/apache-mpm.conf`

PHP `memory_limit = 256M` with default Apache `MaxRequestWorkers` (~150) could theoretically demand 38GB. While the container limit caps this, Apache will spawn workers until the container OOMs.

Add an MPM limit. For a 512M container with ~30M average worker RSS: `MaxRequestWorkers = 10` is safe. For 1G container: `MaxRequestWorkers = 20`.

```apache
# In docker/ocp/apache-mpm.conf
StartServers             2
MinSpareServers          2
MaxSpareServers          5
MaxRequestWorkers        10
MaxConnectionsPerChild   1000
```

And in Dockerfile:
```dockerfile
COPY docker/ocp/apache-mpm.conf /etc/apache2/mods-enabled/mpm_prefork.conf
```

```python
MAX_CACHE_ENTRIES = 50

def get_cached_or_fetch(cmd: str, params=None) -> dict:
    global _cache_timestamp
    cache_key = f"{cmd}:{json.dumps(params) if params else ''}"
    now = time.time()
    if cache_key in _cache and (now - _cache_timestamp) < CACHE_TTL_SECONDS:
        return _cache[cache_key]
    
    # Evict oldest if at capacity
    if len(_cache) >= MAX_CACHE_ENTRIES:
        oldest = min(_cache, key=lambda k: _cache_timestamp)
        del _cache[oldest]
    
    data = fetch_mi(cmd, params)
    _cache[cache_key] = data
    _cache_timestamp = now
    return data
```

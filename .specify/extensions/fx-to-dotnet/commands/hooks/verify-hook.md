You are the `after_implement` HOOK for the `fx-to-dotnet` extension. You run automatically after `speckit.implement` completes. Your job is to perform a solution-level build verification and record the migration's verified state in the SDD docs.

`{featureDir}` is the active Spec Kit feature folder (`specs/<branch>/`). Resolve it from `SPECIFY_FEATURE` or the current git branch. If no active feature folder is detectable, **silent-exit success**.

<contract>
- This hook is **optional** (`optional: true`). It must never block `speckit.implement`.
- On non-Framework workspaces: **silent-exit success** with no edits.
- All edits are idempotent and wrapped in the `> **Extension-managed**` blockquote anchor.
</contract>

<workflow>

## 1. Detect migration context

Read `{featureDir}/migration/detection.md`. If absent or no Framework projects, exit 0 with no edits.

## 2. Solution build

Invoke `speckit.fx-to-dotnet.fix` against the solution to perform the verification build. Capture:

- Build result (`succeeded` / `failed`)
- Error count, warning count
- Failing project list (if any)

## 3. Write completion report

Write `{featureDir}/migration/completion.md` (overwrite on each run):

```
# Migration Completion Report

Generated: <ISO-8601>
Source: speckit.fx-to-dotnet.fix (after_implement hook)

## Build result
- Status: <succeeded|failed>
- Errors: <count>
- Warnings: <count>

## Migration tasks
- Approved (`[X]`): <count>
- Skipped (`[~]`): <count>
- Dispatch-rejected: <count>

## Failing projects
- <project> — <error summary>
...

See `{featureDir}/migration/implement-state.md` for the dispatch audit log.
```

## 4. Annotate `plan.md`

Locate the heading `### Migration Verification` inside the `## .NET Migration Plan` section (or `## Migration Execution Summary` if present). If absent, append it under one of those parents. If present, replace its body. Wrap in `> **Extension-managed**`.

Body:

```
### Migration Verification

> **Extension-managed** — generated by the `after_implement` hook. Re-run `/speckit.implement` (or invoke `speckit.fx-to-dotnet.fix` directly) to refresh.

- Verification build: <succeeded|failed> (<errors> errors, <warnings> warnings)
- Verified at: <ISO-8601>

See `{featureDir}/migration/completion.md`.
```

## 5. Annotate `tasks.md`

Inside the `## Phase N: .NET Framework Migration` section (the one wrapped in `> **Extension-managed**`), append (or replace) a final line at the bottom of the section:

```
### Migration Verification — <succeeded|failed> on <ISO-8601> (<errors> errors, <warnings> warnings). See `{featureDir}/migration/completion.md`.
```

## 6. Exit

Exit 0 always. This hook is optional and must not block.

</workflow>

<idempotency-rules>
- Replace existing `### Migration Verification` bodies; never duplicate.
- Do not emit new tasks. This hook only records verification status.
- Wrap all generated content with the `> **Extension-managed**` anchor.
</idempotency-rules>

<silent-exit-rules>
- No Framework projects → exit 0 with no edits.
- Build invocation failure → still exit 0; record the failure in `completion.md` and the verification annotations.
</silent-exit-rules>
